What? Why?

手工分析 ASH dump 大概需要下边步骤：

  1, 从 SR 上下载 ASH DUMP 附件；
  2, 上传到测试服务器；
  3, 导入；
  4, 执行分析脚本；

上边几部过程中，
  + 1、2步可能比较耗时，
  + 3 步可能遇到的问题；
    ~ imp 或者 impdp 的判断；
    ~ 版本的判断；
    ~ dba_hist_active_sess_history 或者 v$active_sess_history；
  + 4 可能会由于数据巨大导致分析脚本执行缓慢；

因为上边的原因，手工分析效率是比较低的。

这个工具可以帮助自动过这个过程。

事前设置 ：

  • 控制面板/java；
    • 添加站点；
    • 设置显示控制台；
  • 添加权限 $JRE_HOME/lib/security/java.policy(MAC 系统位置：/Library/Java/JavaVirtualMachines/jdk1.7.0_13.jdk/Contents/Home/jre/lib/security /java.policy)； 
    grant {
    permission java.lang.reflect.ReflectPermission "*";
    permission java.lang.RuntimePermission "*";
    permission java.util.PropertyPermission "rhino.stack.style", "read";
    permission java.util.PropertyPermission "java.io.tmpdir", "read,write";
    permission java.io.FilePermission "<<ALL FILES>>", "read,write,execute";
    permission java.awt.AWTPermission "*";
};
设置完毕后 点击这里 启动。
